## 一、产品定位

Cove 是一套面向党政、军工、金融、科研院所、设计单位等强监管领域的**企业级 AI 智能办公平台**。产品形态分为两端：

**Word 插件端**：嵌入 WPS / Microsoft Office 的侧边栏，提供文档校对、翻译、排版、模板沉淀、Skill 调度等核心文档处理能力。

**企业管理后台**：面向系统管理员、安全管理员、审计管理员，提供用户与权限管理、大模型配置、技能与模板分发、知识库接入、安全审计、品牌定制、版本发布等管控能力。

**核心价值主张**：在客户的内网完整闭环地落地"大模型 + 文档生产力"——模型可控、技能可控、数据不出内网、所有改动可回滚可审计。

### 三个核心场景

**党政公文起草、校对、排版**
内置 GB/T 9704-2012 公文排版、国家标准引用校对、公文风格定制包。从文种识别到版记输出全流程覆盖。

**学位 / 学术论文规范化**
内置 GB/T 7713.2-2022 学术论文排版，支持章标题、各级标题、图名、表名、参考文献等细分角色识别，长文档跨章节一致性编排。

**企业模板沉淀与全员复用**
从客户既有 docx 提取格式资产为模板，一键套用到目标文档。模板由后台统一分发，管理员可管控版本和使用范围。

## 三、系统架构

### 插件端架构

插件端采用分层架构，依赖方向单向流动，禁止反向依赖：

```
UI 层（React + Tailwind）
    │
Agent 编排层（模式判定、工具注册、运行期控制）
    │
工具层（LLM 可调用工具、参数校验、任务路由）
    │
WPS 功能封装层（宿主隔离，所有 WPS API 在此封装）
    │
WPS Office 文档
```

**关键设计原则：**

**宿主隔离**：所有 `Application.*` WPS API 调用集中在底层封装层，上层代码不直接依赖 WPS 宿主环境，保证可测试性和可替换性。

**文档处理三阶段流水线**：读取路由（OOXML 快读 / WPS 全文 / 选区）→ LLM 计划生成（按字数+章节切批）→ 重排提交（按原文顺序确认写回），跨批一致性协调器保证长文档不冲突。

**极速排版引擎**：核心排版逻辑在纯规则与程序层面完成（解包 / XML 扫描 / 样式抽取 / 角色规则 / 计划编译 / XML 写回 / 自检），LLM 仅作为低置信场景的增强补充。50 万字级长文档可在 1 分钟内完成。

**确认与回滚体系**：所有修改文档的操作最终汇入快照 + 确认体系。支持修订模式、批注模式、段落对照三种确认弹窗，用户逐条确认或一键回滚。

### 服务端架构

服务端为单二进制部署（Go 语言），同时承载管理后台和客户端 API：

```
管理后台（admin :52001）              客户端 API（client :52000）
    │                                        │
    └──────────────┬─────────────────────────┘
                   │
         业务模块（处理层）
    用户 / 模型 / 技能 / 模板 / 命令
    工具 / 字典 / 标准 / 规则 / DLP
    审计 / 版本 / 知识库 / SSO / 品牌
                   │
             存储层（可选）
    数据库：SQLite（单机）/ PostgreSQL（集群）
    对象存储：本地磁盘 / MinIO / S3
    缓存：Redis（集群模式）
                   │
         外部依赖（管理员配置接入）
    LLM Provider × 9 / 知识库 × 4 / SSO IdP × 3
```

所有变更操作经全局审计中间件全量记录，所有需要 License 鉴权的能力共享同一份验签器。

## 四、部署形态

### 部署对象

- **服务端**（cove-go）：单二进制，同时承载管理后台和客户端 API 服务
- **客户端**（cove-wps）：Word 插件，部署到用户办公机器

### 部署模式

| 配置项 | 单机模式 | 集群模式 |
|--------|---------|---------|
| License 绑定 | 单机设备指纹 | 集群指纹 |
| 数据库 | SQLite（默认） | PostgreSQL 13+ |
| 对象存储 | 本地磁盘（默认） | MinIO / S3 |
| 缓存 | 可选 | 哨兵或集群（必需） |
| 升级方式 | 裸更新 | 滚动部署 |

### 网络架构

- 完全内网部署，无外网依赖
- 客户端轮询获取最新版本，流式下载插件包
- 支持 HTTPS 加密传输，端口最小化
- 支持单向数据流和专网隔离部署

## 五、安全与合规

### 数据安全

- **客户端**：所有持久化经统一存储入口走 WPS 文件系统，不使用浏览器本地存储或 IndexedDB
- **服务端**：用户数据、对话记录、文档资产存储于本地服务器；DB 与对象存储可选加密；备份恢复由部署策略决定

### 访问控制

| 机制 | 说明 |
|------|------|
| JWT 鉴权 | 统一令牌认证 |
| RBAC 权限 | 按角色和操作类型统一校验 |
| 三员分立 | 系统管理员 / 安全管理员 / 审计管理员，可在线切换 |
| SSO | 标准 OIDC + 华信 + 天华三类身份提供商 |
| 组织架构同步 | 与企业的组织架构自动同步 |

### 内容合规

DLP（数据泄露防护）支持关键词和正则两种匹配模式，敏感信息命中后支持三种处置方式：
- **提示**：告知用户存在敏感内容
- **警告**：用户确认后继续
- **拦截**：直接阻止操作

所有命中记录可查可导出。

### 审计追踪

管理员的所有变更操作通过全局审计中间件全量记录，支持多维度筛选查询、备份、还原、清理、保留策略。客户端操作日志通过事件总线流转，关键编辑落入快照，用户可在快照中心回看与回滚。

### License 授权

支持 V1（Ed25519）和 V2（ECDSA P-224）双版本签名。
- 设备指纹绑定，发现绑定不匹配即停服
- 离线激活，适用于内网环境
- 单机 / 集群两种部署模式分别绑定
- 每分钟轻量重加载 + 每小时完整加载，应对手动续期与许可下发
